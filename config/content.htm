Stage3

協同產品設計實習課程中的養成訓練, 必須將各分組的議題內容, 當作是一項全能競賽的訓練.

以十項全能運動為例: 第一天選手必須進行 100 公尺, 跳遠, 鉛球, 跳高與 400 公尺項目, 而第二天則必須進行 110 公尺高欄, 鐵餅, 撐竿跳, 標槍與 1500 米的項目. 每一位選手都必須完成各個項目, 儘管有些選手擅長短跑或田賽項目, 但是全能競賽的總分必須由每一個項目的分數加總.

當各組以兩人, 四人或八人在進行協同產品設計實習專案時, 也必須抱持全能競賽訓練的精神, 讓每一位組員都必須參與各個項目的內容執行, 以本課程各 Stage 的專案內容而言, 共包含: 1) 動態網站維護與轉檔, 2) 倉儲內容改版與同步, 3) 雲端主機內容同步, 4) 提出構想並參與討論, 5) 文獻搜尋, 6) 參數式零組件繪圖, 7) 零組件轉檔, 8) 機電系統模擬與控制, 9) 專案報告編寫與 10) 網際簡報編寫等十個主要項目, 各組在任務執行期間, 應該確實讓每一位組員都實際執行所有任務的練習.

W16

Onshape 零組件繪製 (20%)
零組件必須在 Onshape 繪圖, 並提供可公開分享之零組件連結於頁面及回報區中.
建立 CoppeliaSim 4.1.0 MTB robot 場景 (20%)
請以自行繪製之零件輸入 CoppeliaSim 後, 組合為 MTB robot 模型, 並利用 Leo Editor 以 require 導入 Lua 程式運作, 分別控制各軸轉動示範, 所完成的所有檔案請存入 W16_exam 後壓縮為 W16_exam.7z 後送至個人 @gm 帳號下的 Google Drive 後將連結設為任何人皆可下載, 並將連結與操作嵌入影片放在 W16 頁面與回報區中 .
手臂末端加入 components-gripper-suction pad 吸盤 (20%)
請接續上述 MTB robot, 在其末端接上 force sensor 後接上標準 suction pad 後, 分別導入 W15 週線上課程之鍵盤控制程式後, 拍攝影片示範 W15 線上課程中之操作與示範過程.
逆向運動學函式 (20%)
請根據 W15 線上課程內容之 Inverse Kinematics 方程式 (影片1 或影片2), 以程式指定方塊取放之兩個位置 - (0.2, 0.7, 0.05) 與 (-0.3, -0.55, 0.05), 以前述 Leo Editor Lua 程式編寫方法, 分別採鍵盤控制與程式迴圈方式執行 W15 兩個指定位置之方塊取放.
Python remote API 逆向運動學函式 (20%)
請利用 Python remote API 程式重現以迴圈方式執行 W15 兩個指定位置之方塊取放.
